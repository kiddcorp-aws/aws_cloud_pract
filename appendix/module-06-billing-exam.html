AWS Pricing Fundamentals

Understanding how AWS charges for services

Let us start with the foundational pricing concepts. Understanding these principles will help you both manage costs in real-world scenarios and answer exam questions.

AWS Pricing Philosophy

Pay-As-You-Go

Pay only for the resources you actually consume. No upfront commitments or long-term contracts required.

Save When You Reserve

Commit to a certain level of usage (1 or 3 years) and receive significant discounts compared to On-Demand pricing.

Pay Less as You Use More

Volume-based discounts are applied automatically. The more you use, the lower the per-unit cost becomes.

These three pillars define the AWS pricing model and are frequently tested on the exam.

AWS pricing is built on three pillars. Pay-as-you-go removes the need for large upfront investments. Reservations reward predictable workloads. Volume discounts reward scale. Candidates should know all three for the exam.

AWS Free Tier

Always Free

  • Lambda: 1M requests/month
  • DynamoDB: 25 GB storage
  • CloudWatch: 10 custom metrics
  • SNS: 1M publishes

12 Months Free

  • EC2: 750 hrs/month (t2.micro or t3.micro)
  • S3: 5 GB standard storage
  • RDS: 750 hrs/month (db.t2.micro)
  • CloudFront: 1 TB data transfer out

Trials

  • Short-term free trial offers
  • Start from activation date
  • Examples: Amazon SageMaker, Inspector, GuardDuty
Important: The 12 Months Free tier begins from the date your AWS account is created, not from the date you first use the service.
The Free Tier is a common exam topic. Always Free offers never expire. The 12 Months Free category starts from account creation date, which is an important distinction. Trials are service-specific and vary in duration.

Key Pricing Drivers

Compute

  • Charged per hour or per second
  • Varies by instance type and size
  • OS licensing may add cost

Storage

  • Charged per GB stored per month
  • Varies by storage class/type
  • Provisioned IOPS costs extra

Data Transfer

  • Inbound data transfer: FREE
  • Outbound data transfer: Charged per GB
  • Between Regions: Charged both directions
  • Same AZ: Free (using private IPs)

Data transfer IN to AWS is always free. Data transfer OUT is charged.

There are three key pricing drivers: compute, storage, and data transfer. The most important rule to remember is that inbound data transfer is free, but outbound is charged. Data transfer between Availability Zones and Regions also incurs charges. This is a frequent exam topic.

EC2 Pricing Models

Model Commitment Discount Best For
On-Demand None None (baseline) Short-term, unpredictable workloads
Reserved Instances 1 or 3 years Up to 72% Steady-state, predictable usage
Savings Plans 1 or 3 years ($/hr) Up to 72% Flexible across instance families
Spot Instances None Up to 90% Fault-tolerant, flexible workloads
Dedicated Hosts On-Demand or Reserved Varies Compliance, licensing requirements
EC2 pricing models are a must-know for the exam. On-Demand is the most flexible but most expensive. Reserved Instances and Savings Plans offer significant discounts for commitments. Spot offers the deepest discounts but instances can be interrupted. Dedicated Hosts are for compliance or licensing needs.

S3 and Lambda Pricing

S3 Pricing Factors

  • Storage class: Standard, IA, Glacier, etc.
  • Amount of data stored (per GB/month)
  • Number of requests: PUT, GET, LIST
  • Data transfer out of S3
  • Lifecycle transitions between classes
S3 Intelligent-Tiering automatically moves data to the most cost-effective tier.

Lambda Pricing

  • Number of requests: First 1M free/month
  • Duration: Measured in ms
  • Memory allocated: Affects cost per ms
Lambda pricing formula:
Cost = (Requests x $0.20/1M) + (GB-seconds x $0.0000166667)
S3 pricing depends on storage class, the volume of data, requests, and data transfer. Lambda pricing is based on the number of requests and duration, measured in milliseconds. Both services have generous free tier allocations. Understand these pricing components for the exam.

Billing & Cost Management Tools

Monitor, analyze, and optimize your AWS spending

Now let us look at the tools AWS provides for managing and optimizing costs. These tools are frequently tested on the exam and are essential for real-world AWS usage.

AWS Billing Dashboard & Cost Explorer

Billing Dashboard

  • High-level overview of month-to-date spending
  • Summary of charges by service
  • Forecasted end-of-month charges
  • Access to all billing and cost tools

AWS Cost Explorer

  • Visualize and understand costs over time
  • Filter by service, Region, tag, or account
  • Forecast future costs (up to 12 months)
  • Identify cost trends and anomalies
  • Provides Reserved Instance recommendations

Cost Explorer is your primary tool for analyzing historical spending and forecasting future costs.

The Billing Dashboard gives a quick snapshot of current spending. Cost Explorer is the more powerful tool for visualizing costs over time, filtering by dimensions, and getting RI purchase recommendations. The exam often asks about which tool to use for specific scenarios.

AWS Budgets

Set Custom Budgets and Get Alerts

  • Cost Budgets: Track spending against a dollar amount
  • Usage Budgets: Track usage of specific services
  • Reservation Budgets: Track RI/Savings Plan utilization
  • Savings Plans Budgets: Track coverage and utilization
Alert Thresholds:
  • Set alerts at actual or forecasted amounts
  • Notify via email or SNS topic
  • Trigger automated actions (e.g., apply IAM policy to restrict launches)

AWS Budgets is the tool for setting proactive spending limits and alerts.

AWS Budgets allows you to set custom budgets and receive alerts when actual or forecasted spending exceeds your thresholds. You can set up cost, usage, reservation, and savings plans budgets. Alerts can trigger emails, SNS notifications, or automated actions. This is distinct from Cost Explorer, which is for analysis.

More Cost Management Tools

AWS Cost and Usage Reports (CUR)

  • Most comprehensive cost and usage data
  • Granular, line-item billing data
  • Delivered to S3 bucket
  • Can be analyzed with Athena, QuickSight, or Redshift
  • Includes resource-level detail

AWS Pricing Calculator

  • Estimate costs before deploying
  • Build cost estimates for solutions
  • Compare pricing across Regions
  • Generate shareable estimate links
  • Available at calculator.aws
Exam tip: CUR is the most detailed billing report. Pricing Calculator is for pre-deployment cost estimation. Know the difference.
The Cost and Usage Reports provide the most detailed billing data, delivered to an S3 bucket for analysis. The Pricing Calculator at calculator.aws is used to estimate costs before deploying workloads. The exam will test whether you know which tool is appropriate for pre-deployment estimates versus post-deployment analysis.

Consolidated Billing & Cost Allocation Tags

Consolidated Billing (AWS Organizations)

  • Single payment method for all accounts
  • Combined usage across accounts
  • Volume pricing discounts applied to aggregate
  • Detailed billing reports per account
  • Free feature of AWS Organizations

Cost Allocation Tags

  • Label resources for cost tracking
  • AWS-generated tags: Automatically applied (e.g., aws:createdBy)
  • User-defined tags: Custom labels (e.g., Department, Project)
  • Must be activated in Billing console
  • Appear in Cost Explorer and CUR

Consolidated billing aggregates usage across accounts, enabling volume discounts.

Consolidated billing is a key benefit of AWS Organizations. By combining usage, organizations can reach higher volume discount tiers. Cost allocation tags allow you to categorize and track costs by business dimensions like project, department, or environment. Both are commonly tested topics.

AWS Support Plans

Choosing the right level of support for your organization

AWS offers multiple support plan tiers. Understanding the differences, especially response times and included features, is essential for the exam.

AWS Support Plan Tiers

Feature Basic Developer Business Enterprise On-Ramp Enterprise
Price Free $29+/mo $100+/mo $5,500+/mo $15,000+/mo
Tech Support None Business hours email 24/7 phone, email, chat 24/7 phone, email, chat 24/7 phone, email, chat
Response (Critical) N/A 12 hours (general) 1 hour 30 minutes 15 minutes
TAM No No No Pool of TAMs Designated TAM
Trusted Advisor 7 core checks 7 core checks Full checks Full checks Full checks
The support plan comparison table is heavily tested on the exam. Key differentiators include: a TAM (Technical Account Manager) is only available at Enterprise On-Ramp and Enterprise levels. Full Trusted Advisor checks start at Business tier. Critical response times vary from 1 hour at Business to 15 minutes at Enterprise.

Choosing the Right Support Plan

Basic: Experimenting or testing AWS. No technical support cases needed.
Developer: Development and testing environments. One primary contact needs email support during business hours.
Business: Production workloads. Need 24/7 phone support and full Trusted Advisor. Most common for production use.
Enterprise On-Ramp: Business-critical workloads. Need a pool of TAMs and 30-minute critical response.
Enterprise: Mission-critical workloads. Need a designated TAM, 15-minute critical response, and concierge support.
When choosing a plan, match it to the workload criticality. Basic for experimentation, Developer for dev and test, Business for production, Enterprise On-Ramp and Enterprise for business-critical and mission-critical workloads respectively. The exam may present scenarios and ask which plan is appropriate.

Health Dashboard & Trusted Advisor

AWS Health Dashboard

  • Personalized view of AWS service health
  • Alerts for events affecting your resources
  • Proactive notifications for scheduled changes
  • Remediation guidance included
  • Available to all support plans

AWS Trusted Advisor

  • Automated best-practice checks across 5 categories:
  • Cost Optimization -- reduce spending
  • Performance -- improve speed
  • Security -- close gaps
  • Fault Tolerance -- increase resilience
  • Service Limits -- avoid hitting quotas
Exam tip: Basic and Developer plans get only 7 core Trusted Advisor checks (mostly security). Full checks require Business tier or above.
The AWS Health Dashboard provides personalized alerts about AWS events that may affect your account. Trusted Advisor performs automated checks across five categories. An important exam distinction: Basic and Developer plans only get 7 core checks, while Business, Enterprise On-Ramp, and Enterprise get the full set of checks.

Exam Preparation -- CLF-C02

Strategies, domains, and practice for certification success

Now we transition to dedicated exam preparation. This section covers the exam structure, domain breakdown, strategies, and sample questions to prepare you for the CLF-C02 certification.

CLF-C02 Exam Overview

Format

  • 65 questions total
  • Multiple choice (1 correct) and multiple response (2+ correct)
  • 50 scored, 15 unscored

Logistics

  • 90 minutes duration
  • Testing center or online proctored
  • Valid for 3 years

Scoring

  • Scaled score: 100-1000
  • Passing score: 700
  • No penalty for guessing

There is no penalty for guessing -- always answer every question.

The CLF-C02 exam has 65 questions, but only 50 are scored. You have 90 minutes. The passing score is 700 out of 1000 on a scaled score. There is no penalty for wrong answers, so always answer every question even if you are unsure. The exam can be taken at a testing center or via online proctoring.

Exam Domains & Weights

Domain 1: Cloud Concepts

24% of exam

Value proposition, cloud economics, cloud architecture design principles

Domain 2: Security and Compliance

30% of exam

Shared responsibility, governance, compliance, access management

Domain 3: Cloud Technology and Services

34% of exam

Deploying, managing services, global infrastructure, networking

Domain 4: Billing, Pricing, and Support

12% of exam

Pricing models, billing management, support resources

Key insight: Security (30%) and Technology (34%) together make up 64% of the exam. Focus your study time accordingly.
The four domains have different weights. Domain 3, Cloud Technology and Services, carries the highest weight at 34%. Domain 2, Security and Compliance, is close behind at 30%. Together they represent nearly two-thirds of the exam. Domain 4, which covers billing and pricing, is only 12% but is still important for passing.

This slide lists the key services and concepts for each domain. For Cloud Concepts, focus on the Well-Architected Framework and cloud benefits. Security revolves around IAM and the Shared Responsibility Model. Technology covers the core compute, storage, database, and networking services. Exam Question Strategies

Answering Techniques

  • Read the entire question before looking at answers
  • Identify keywords: "most cost-effective," "most secure," "least operational overhead"
  • Eliminate obviously wrong answers first
  • If two answers seem correct, choose the more AWS-native solution
  • Flag and revisit difficult questions

Time Management

  • ~83 seconds per question on average
  • Do not spend more than 2 minutes on any one question
  • Answer easy questions first, flag harder ones
  • Review flagged questions at the end
  • Never leave a question blank

When in doubt, choose the answer that is the most managed, most scalable, and most AWS-native.

These strategies will help during the exam. Look for qualifier keywords in the question that guide you to the correct answer. Elimination is your most powerful technique for multiple choice. AWS generally favors managed, serverless, and native solutions over self-managed approaches.

Sample Exam Questions

Question 1

A company wants to receive alerts when their AWS spending exceeds a defined threshold. Which AWS service should they use?

  • AWS Cost Explorer
  • AWS Budgets
  • AWS Pricing Calculator
  • AWS Cost and Usage Reports
Answer: B -- AWS Budgets
AWS Budgets allows you to set custom spending thresholds and receive alerts when actual or forecasted costs exceed those thresholds. Cost Explorer is for analysis, not alerts.
This is a typical exam question that tests your knowledge of which cost management tool to use. The key word here is "alerts" -- which maps directly to AWS Budgets. Cost Explorer analyzes costs but does not send alerts on thresholds. The Pricing Calculator is for pre-deployment estimates.

Sample Exam Questions

Question 2

Which AWS Support plan provides access to a designated Technical Account Manager (TAM)?

  • Basic
  • Business
  • Enterprise On-Ramp
  • Enterprise
Answer: D -- Enterprise
Only the Enterprise plan provides a designated TAM. Enterprise On-Ramp provides access to a pool of TAMs, not a designated one. Business and Basic do not include TAM access.
This question tests the distinction between Enterprise and Enterprise On-Ramp. The keyword is "designated" TAM. Enterprise On-Ramp provides a pool of TAMs, but only Enterprise provides a single designated TAM. This is a frequently tested distinction.

Sample Exam Questions

Question 3

Under the AWS Shared Responsibility Model, which of the following is the customer's responsibility?

  • Physical security of data centers
  • Patching the hypervisor
  • Configuring security group rules
  • Maintaining network infrastructure hardware
Answer: C -- Configuring security group rules
Security groups are customer-managed network firewalls. Physical security, hypervisor patching, and network hardware are all AWS responsibilities.
The Shared Responsibility Model is the most commonly tested topic on the exam. Customers are responsible for security IN the cloud, which includes configuring security groups, managing IAM, encrypting data, and patching guest operating systems. AWS handles security OF the cloud, including physical infrastructure, hypervisors, and managed services.

Sample Exam Questions

Question 4

A company needs to run a workload that can tolerate interruptions and wants the lowest possible cost. Which EC2 purchasing option should they choose?

  • On-Demand Instances
  • Reserved Instances
  • Spot Instances
  • Dedicated Hosts
Answer: C -- Spot Instances
Spot Instances offer up to 90% discount compared to On-Demand pricing but can be interrupted with a 2-minute warning. They are ideal for fault-tolerant workloads.
This question combines two keywords: "tolerate interruptions" and "lowest possible cost." Both point directly to Spot Instances. The exam frequently tests whether you can match workload requirements to the correct EC2 pricing model. If the workload can handle interruptions, Spot is always the most cost-effective option.
