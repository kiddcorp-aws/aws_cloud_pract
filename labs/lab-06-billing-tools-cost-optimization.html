Lab 6: AWS Billing Tools and Cost Optimization

Duration: 45 minutes

Level: Introductory

Prerequisites: Completed Labs 1-5; AWS console access

Lab Objectives

  • Explore the AWS Billing and Cost Management Dashboard
  • Use the AWS Pricing Calculator to estimate a solution's cost
  • Compare pricing across multiple architectures (EC2 vs. serverless)
  • Create both cost-based and usage-based AWS Budgets with alert thresholds
  • Review AWS Trusted Advisor recommendations
  • Explore Cost Explorer to analyze spending patterns
  • Configure cost allocation tags for tracking spend by project and team
  • Compare AWS Support plans and understand their features
  • Review AWS Free Tier categories and monitor usage allowances
Note: Some billing features may be restricted in lab/training accounts. If you encounter permission errors, your instructor will demonstrate the step. The AWS Pricing Calculator (Part B) is a public tool and requires no account permissions.

Part A: Explore the Billing Dashboard

1 In the AWS Management Console, click your Account name in the top-right corner and select Billing and Cost Management.

2 On the Billing Dashboard home page, review the following:

  • Month-to-date spend — current charges for the billing period
  • Forecasted month-end charges — AWS's projection based on current usage
  • Top services by cost — which services are driving spend

3 In the left navigation, click Bills. Expand a service to see the detailed line items that make up the charge (e.g., EC2 hours, EBS storage, data transfer).

4 Click Free Tier in the left navigation. Review which Free Tier services you have used and how much of each allowance remains. Note the usage percentages.

18 Review the available checks across the five categories. For each category, note the status indicators:

  • Performance — overutilized resources, service limits approaching

Part G: Cost Allocation Tags

Why tags matter: Cost allocation tags allow organizations to categorize AWS costs by project, team, environment, or any custom dimension. Without tags, all costs appear as a single line item per service, making it difficult to understand who is spending what.

26 Navigate to Billing and Cost Management > Cost allocation tags in the left navigation.

27 Review the two types of cost allocation tags:

  • AWS-generated tags: Automatically created by AWS (e.g., aws:createdBy). These track which IAM user or role created a resource.
  • User-defined tags: Tags you applied to resources in earlier labs (e.g., Environment=Lab, Name=my-instance).

If you see any user-defined tags listed (from resources tagged in earlier labs), select them and click Activate. Note that it can take up to 24 hours for activated tags to appear in cost reports.

Note: If no user-defined tags appear, this is normal for new accounts. The key concept is understanding that tags must be activated here before they appear in Cost Explorer reports. You can skip activation and continue to the next step.

28 Navigate to Cost Explorer. In the grouping options, look for Tag as a group-by dimension. If you have activated tags with associated cost data, try grouping by a tag such as Environment or Name.

29 Discussion: Consider how an organization would use a tagging strategy in practice:

  • Project — track costs per application or initiative (e.g., Project=WebApp)
  • Team — attribute costs to the team responsible (e.g., Team=Backend)
  • Environment — separate development, staging, and production costs (e.g., Environment=Production)
  • CostCenter — map costs to internal accounting codes

A well-enforced tagging policy is one of the most important cost management practices in AWS. Many organizations use AWS Organizations Service Control Policies (SCPs) or tag policies to enforce mandatory tags on all resources.

Part H: Create a Usage-Based Budget

Why usage budgets? Cost budgets alert you when spending exceeds a dollar threshold. Usage budgets alert you when consumption of a specific resource type exceeds a threshold — useful for staying within Free Tier limits or tracking resource utilization independently of cost.

30 Navigate to Billing and Cost Management > Budgets.

31 Click Create a budget and select Customize (advanced) > Usage budget. Click Next.

32 Configure the usage budget:

  • Budget name: lab-ec2-hours-budget
  • Period: Monthly
  • Budget renewal type: Recurring
  • Usage type group: Select EC2: Running Hours (or the closest available option for EC2 instance hours)
  • Budgeted amount: 750 hours (this matches the Free Tier monthly limit for a single t2.micro/t3.micro instance)
  • Click Next

33 Add an alert threshold:

  • Threshold: 80% of budgeted amount (Actual usage)
  • Email recipients: Enter your email address
  • Click Next, review, and Create budget

34 Review the Budgets dashboard. You should now see both budgets:

  • lab-monthly-budget — tracks dollar spend (from Part C)
  • lab-ec2-hours-budget — tracks EC2 running hours (just created)

Discussion: When would a usage budget be more useful than a cost budget? Consider a scenario where you are within Free Tier — cost is $0, but usage is climbing toward the Free Tier limit. A cost budget would never trigger, but a usage budget would alert you before you start incurring charges.

38 Discussion: Which support plan would you recommend for each of the following scenarios?

42 Discussion: How do you avoid accidental charges after the Free Tier period ends?

  • Set up usage budgets (as you did in Part H) to alert before exceeding Free Tier limits
  • Enable Free Tier usage alerts in Billing Preferences (Billing > Preferences > check "Receive Free Tier Usage Alerts")
  • Regularly review the Free Tier dashboard to monitor consumption
  • Terminate or delete resources you are no longer using — stopped EC2 instances still incur EBS storage charges
  • Be aware that some services (like NAT Gateways and Elastic IPs not attached to running instances) are not part of the Free Tier and incur charges immediately
