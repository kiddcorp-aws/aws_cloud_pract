Lab 6: AWS Billing Tools and Cost Optimization

Duration: 15 minutes

Level: Introductory

Prerequisites: Completed Labs 1-5; AWS console access

Lab Objectives

  • Explore the AWS Billing and Cost Management Dashboard
  • Use the AWS Pricing Calculator to estimate a solution's cost
  • Create an AWS Budget with alert thresholds
  • Review AWS Trusted Advisor recommendations
  • Explore Cost Explorer to analyze spending patterns
Note: Some billing features may be restricted in lab/training accounts. If you encounter permission errors, your instructor will demonstrate the step. The AWS Pricing Calculator (Part B) is a public tool and requires no account permissions.

Part A: Explore the Billing Dashboard

1 In the AWS Management Console, click your Account name in the top-right corner and select Billing and Cost Management.

2 On the Billing Dashboard home page, review the following:

  • Month-to-date spend — current charges for the billing period
  • Forecasted month-end charges — AWS's projection based on current usage
  • Top services by cost — which services are driving spend

3 In the left navigation, click Bills. Expand a service to see the detailed line items that make up the charge (e.g., EC2 hours, EBS storage, data transfer).

4 Click Free Tier in the left navigation. Review which Free Tier services you have used and how much of each allowance remains. Note the usage percentages.

Part B: AWS Pricing Calculator

Scenario: Your company wants to estimate the monthly cost of running a simple web application on AWS with the following architecture: one EC2 instance, an Application Load Balancer, an RDS database, and an S3 bucket for static assets.

5 Open the AWS Pricing Calculator (this is a public web tool — no sign-in required). Your instructor will provide guidance on accessing it.

6 Click Create estimate. Add the following services one at a time:

7 Add Amazon EC2:

  • Region: US East (N. Virginia)
  • Operating System: Linux
  • Instance type: t3.medium
  • Pricing model: On-Demand
  • Quantity: 1 instance, running 24/7 (730 hours/month)
  • Storage: 30 GB gp3 EBS volume

Note the estimated monthly cost, then click Add to my estimate.

8 Add Elastic Load Balancing:

  • Type: Application Load Balancer
  • Region: US East (N. Virginia)
  • Number of ALBs: 1
  • Processed bytes: 10 GB/month

Note the estimated monthly cost, then click Add to my estimate.

9 Add Amazon RDS:

  • Engine: MySQL
  • Region: US East (N. Virginia)
  • Instance type: db.t3.micro
  • Deployment: Single-AZ
  • Storage: 20 GB gp3
  • Pricing model: On-Demand

Note the estimated monthly cost, then click Add to my estimate.

10 Add Amazon S3:

  • Region: US East (N. Virginia)
  • Storage class: S3 Standard
  • Storage amount: 50 GB/month
  • GET requests: 100,000/month
  • PUT requests: 10,000/month

Note the estimated monthly cost, then click Add to my estimate.

11 Review your total estimate. Record the results:

Service Monthly Cost (USD)
Amazon EC2 (t3.medium + EBS)
Application Load Balancer
Amazon RDS (db.t3.micro)
Amazon S3 (50 GB Standard)
Total Estimated Monthly Cost

12 Challenge: Go back to the EC2 configuration and change the pricing model from On-Demand to a 1-year Reserved Instance (No Upfront). Note how much the monthly cost decreases. Calculate the percentage savings.

Part C: Create an AWS Budget

13 Back in the AWS Console, navigate to Billing and Cost Management > Budgets in the left navigation.

14 Click Create a budget and select Customize (advanced) > Cost budget. Click Next.

15 Configure the budget:

  • Budget name: lab-monthly-budget
  • Period: Monthly
  • Budget renewal type: Recurring
  • Budgeted amount: $10.00
  • Click Next

16 Add an alert threshold:

  • Threshold: 80% of budgeted amount (Actual cost)
  • Email recipients: Enter your email address
  • Click Next, review, and Create budget

Part D: Review Trusted Advisor

17 Navigate to Trusted Advisor (search for it in the console search bar).

18 Review the available checks across the five categories. For each category, note the status indicators:

  • Cost Optimization — idle resources, underutilized instances
  • Performance — overutilized resources, service limits approaching
  • Security — open security groups, MFA on root, public S3 buckets
  • Fault Tolerance — backup configurations, Multi-AZ usage
  • Service Limits — how close you are to AWS service quotas

19 Expand the Security category. Look for any findings related to:

  • Security groups with unrestricted access (0.0.0.0/0)
  • MFA on root account
  • IAM password policy strength

Note which checks show green (no issues), yellow (investigation recommended), or red (action recommended).

Note: With the Basic and Developer support plans, only a limited set of Trusted Advisor checks are available. Business and Enterprise plans unlock all checks.

Part E: Explore Cost Explorer

20 Navigate back to Billing and Cost Management > Cost Explorer.

21 If Cost Explorer is enabled, explore the following views:

  • Change the time range to Last 3 months
  • Group by Service to see which services cost the most
  • Group by Region to see cost distribution across regions
  • Try the Forecast view to see projected future costs

Part F: Clean Up

22 Delete the budget created in Part C:

  • Navigate to Budgets
  • Select lab-monthly-budget and click Delete

Lab Complete

You have explored the AWS Billing Dashboard, used the Pricing Calculator to estimate a multi-service architecture, created a budget with alert thresholds, reviewed Trusted Advisor security and cost recommendations, and analyzed spending patterns in Cost Explorer. These are essential skills for managing AWS costs in any organization.

Discussion Questions

  1. How much did switching from On-Demand to Reserved Instance pricing save in your estimate? When would this trade-off not make sense?
  2. Why is it important to set up budget alerts before deploying production workloads?
  3. Which Trusted Advisor category do you think provides the most immediate value for a new AWS account?
  4. How could cost allocation tags help a company with multiple teams sharing a single AWS account?
  5. What is the difference between the Billing Dashboard, Cost Explorer, and AWS Budgets — when would you use each?
