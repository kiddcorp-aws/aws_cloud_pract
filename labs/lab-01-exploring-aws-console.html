Lab 1: Exploring the AWS Management Console

Duration: 45 minutes

Level: Introductory

Prerequisites: AWS account credentials provided by your instructor

Lab Objectives

  • Sign in to the AWS Management Console
  • Navigate the console and locate key services
  • Explore the Region selector and understand service availability
  • Review your account's billing dashboard
  • Use CloudShell to query AWS services via the CLI
  • Work with Resource Groups and Tags
  • Explore AWS Support and Documentation resources
Important: Use only the credentials provided by your instructor. Do not create resources outside the scope of this lab to avoid unexpected charges.

Part A: Signing In and Console Overview

1 Open your web browser and navigate to the AWS Management Console sign-in page. Your instructor will provide the URL and credentials.

2 Sign in using the IAM user credentials provided (Account ID, Username, Password).

3 Once signed in, take a moment to explore the console home page. Confirm you can see the search bar at the top, your account name in the top-right corner, and the Region selector dropdown next to it. Note the following areas:

  • Search bar at the top — type a service name to quickly navigate
  • Recently visited services — shows your most-used services
  • Favorites — pin services you use frequently (star icon)
  • Region selector — dropdown in the top-right navigation bar
  • Account menu — top-right corner with your account info

Part B: Navigating Services and Regions

4 Click the Region selector dropdown. Note the available regions. Switch to us-east-1 (N. Virginia) — this is the most commonly used region for learning.

5 Use the search bar to find and navigate to each of the following services. For each one, check whether the Region selector in the top-right shows a specific Region (regional service) or says "Global" (global service). Record your findings.

  • EC2 — Elastic Compute Cloud
  • S3 — Simple Storage Service
  • IAM — Identity and Access Management
  • VPC — Virtual Private Cloud
  • CloudWatch — Monitoring and Observability
  • RDS — Relational Database Service

6 Type IAM in the search bar and click IAM from the results. Notice that IAM is a Global service — the region selector shows "Global" instead of a specific region. This means IAM resources are not region-specific.

7 Type S3 in the search bar and click S3 from the results. Notice that S3 also shows as a global console, though buckets are created in specific regions.

Part C: Exploring Account and Billing

8 Click on your Account name in the top-right corner and select Billing and Cost Management (if your permissions allow).

9 Explore the Billing Dashboard. Complete the following:

  • Record the current month-to-date cost
  • Identify the top service by cost
  • Check what percentage of Free Tier usage has been consumed

Part D: Using CloudShell

10 Click the CloudShell icon in the top navigation bar (terminal icon). Wait for it to initialize.

11 In the CloudShell terminal, run the following commands:

  • aws sts get-caller-identity — shows your current IAM identity
  • aws s3 ls — lists S3 buckets in the account
  • aws ec2 describe-regions --output table — lists all available regions

Part E: Explore Services with CloudShell

12 Return to CloudShell (click the terminal icon in the top navigation bar if you closed it). Run the following command to list the VPCs in your account:

  • aws ec2 describe-vpcs --output table

Note the VPC IDs and CIDR blocks displayed. Every AWS account comes with a default VPC in each region.

13 List the IAM users configured in your account:

  • aws iam list-users --output table

You should see at least the user you are currently signed in as. Note the UserName and CreateDate columns.

14 List the S3 bucket names in the account using a JMESPath query:

  • aws s3api list-buckets --query "Buckets[].Name"

Compare this output to the aws s3 ls command you ran earlier. The --query parameter uses JMESPath to filter JSON output — a useful technique you will use throughout this course.

15 List the Availability Zones in the us-east-1 region:

  • aws ec2 describe-availability-zones --region us-east-1 --output table

Note how many Availability Zones are available and their zone names (e.g., us-east-1a, us-east-1b, etc.).

16 Now compare by listing the Availability Zones in a different region:

  • aws ec2 describe-availability-zones --region us-west-2 --output table

Compare the number of Availability Zones and their naming convention with the us-east-1 output. Different regions have different numbers of AZs, which affects how you design for high availability.

Part F: Resource Groups and Tagging

17 In the console search bar, type Resource Groups and select Resource Groups & Tag Editor. This service helps you organize and find AWS resources using tags.

18 In the left navigation pane, click Tag Editor. Configure the search with the following settings:

  • Regions: Select All regions
  • Resource types: Select All supported resource types

Click Search resources. Review the results — these are all the taggable resources currently in your account. Note the resource types and any existing tags.

19 Navigate to S3 in a new tab. If you have any S3 buckets, click on one. If not, navigate to VPC > Your VPCs and click on the default VPC. Click the Tags tab and click Manage tags (or Add/Edit tags). Add a tag:

  • Key: Environment
  • Value: Lab

Click Save to apply the tag.

20 Return to Tag Editor. This time, search for resources with a specific tag. Configure the search:

  • Regions: Select All regions
  • Resource types: Select All supported resource types
  • Tags: Add a tag filter with Key = Environment and Value = Lab

Click Search resources. You should see only the resource you just tagged. This demonstrates how tags let you filter and organize resources across your entire AWS account.

21 Add a second tag to the same resource: Key = Owner, Value = your-name. Then return to Tag Editor and search by tag Key = Owner. Verify your resource appears. Tags are used in practice for:

  • Cost allocation — track spending by project, team, or environment
  • Automation — scripts can target resources by tag
  • Access control — IAM policies can restrict actions based on resource tags
  • Organization — quickly find and group related resources across services and regions

Part G: Explore AWS Support and Documentation

22 Click the ? (question mark) icon in the top navigation bar, or use the search bar to navigate to Support Center. Note which support plan is active (e.g., Basic).

23 Click Create case to see the case types available (Account and billing, Technical), then click Cancel without creating a case. Review what is included with the Basic support plan:

  • Access to documentation and whitepapers
  • AWS Trusted Advisor (limited checks)
  • AWS Personal Health Dashboard
  • Community forums

In production environments, organizations often upgrade to Developer, Business, or Enterprise support for faster response times and technical account management.

24 Open the AWS Documentation by navigating to docs.aws.amazon.com (or click Documentation from the Support menu). Search for S3 and browse the Amazon S3 User Guide. Note how documentation is organized into sections like Getting Started, User Guide, API Reference, and CLI Reference.

25 Find the Amazon S3 FAQ page by navigating to aws.amazon.com/s3/faqs/. FAQ pages are an excellent resource for understanding service capabilities, limits, and pricing — and they are frequently referenced in the AWS Cloud Practitioner exam.

26 Navigate to the AWS Service Health Dashboard by going to health.aws.amazon.com/health/status (or search for Health Dashboard in the console). Check whether all services show a green checkmark for the us-east-1 region. Click on any service name to see its event history. The Health Dashboard provides real-time visibility into service availability and is the first place to check when you suspect an AWS service disruption.

Lab Complete

You have successfully navigated the AWS Management Console, explored key services, reviewed regions, used CloudShell to query AWS resources via the CLI, organized resources with tags, and explored AWS Support and Documentation. These skills form the foundation for all subsequent labs in this course.
